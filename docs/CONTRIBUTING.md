# Contributing to Sudarshan

First off, thank you for considering contributing to Sudarshan! It's people like you that make Sudarshan such a great tool for the mobile security community.

## Where do I go from here?

If you've noticed a bug or have a feature request, please [open an issue](https://github.com/your-org/sudarshan/issues) first. 

## Setting up your environment

1. Fork the repo and create your branch from `main`.
2. Follow the setup guide in [HOW_TO_RUN.md](HOW_TO_RUN.md) to get your local environment running.
3. Ensure you have the `black` and `flake8` linters installed for Python, and `eslint` for React.

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. Your code must pass all tests before it can be merged. Run `pytest` locally to ensure no tests are broken.
4. You may merge the Pull Request in once you have the sign-off of two other developers, or if you do not have permission to do that, you may request the second reviewer to merge it for you.

## Code Style

- **Python**: We strictly adhere to PEP-8. Use `black` to format your code.
- **JavaScript/React**: Use Prettier for formatting and ESLint for linting.
- **Commit Messages**: Use conventional commits (e.g., `feat: added AI agent for malware analysis`, `fix: resolving async db lock`).

## Any questions?
Reach out to the maintainers on our Discord or via GitHub Discussions.
